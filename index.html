01 — Visão Geral

O que é o KelTec Media Play?

KelTec Media Play Xtream é um add-on para Kodi 21 (Omega) desenvolvido do zero em Python 3, que integra listas IPTV no padrão Xtream Codes API com metadados ricos do TMDB (The Movie Database), guia de programação EPG/XMLTV e um proxy reverso HLS de alta resiliência baseado no F4MTESTER. Distribuído gratuitamente para usuários do mundo inteiro, com servidores gerenciados remotamente via JSON seguro.

📡

TV ao Vivo

Canais IPTV com stream HLS/M3U8 via proxy F4M com cache e reconexão automática. Buffer inteligente evita quedas de sinal.

🎬

Filmes & Séries

VOD completo com metadados TMDB: poster HD, fanart, plot em português, elenco, diretor, rating e gênero.

📺

EPG Integrado

Guia de programação XMLTV com atualização automática por dia. Mostra o programa atual e o próximo em cada canal.

🌐

Multi-Servidor

Até 10+ servidores gerenciados remotamente via JSON. Fallback automático quando um servidor cai, sem expor credenciais.

🔒

Segurança Total

Logs controlados por debug_mode. Erros genéricos ao usuário. URLs, usuários e senhas jamais expostos nos logs do Kodi.

🎭

TMDB Live

Metadados do programa atual em canais ao vivo. Cruzamento automático EPG + TMDB para poster e plot do show.

🎨

Modos Visuais

Suporte a todos os modos do Kodi: lista, grade, miniaturas, poster view. Compatible com qualquer skin.

🔄

DNS Customizado

Resolver DNS próprio com cache de 4h usando OpenDNS, Cloudflare e Google DNS. Contorna bloqueios de ISP.

🛡️

Proxy HTTP

Suporte opcional a proxies HTTP rotativos para mascarar origem das requisições via lista pública scraped.

03 — Módulos

Descrição dos Módulos

Arquivo Função Principal Tecnologia Status
main.py Núcleo do add-on. Roteamento de modos, menus, EPG, reprodução, TMDB, múltiplos servidores. Python 3 · Kodi API · xbmcplugin CORE
server_loader.py Carrega servidores de JSON remoto via parser próprio (sem json.loads), com cache e suporte a debug_mode. Python 3 · re · requests CORE
tmdb_helper.py Integração com TMDB API v3. Busca filmes, séries, episódios. Cache local JSON. Retorna metadados formatados para Kodi. TMDB API v3 · requests · JSON cache INTEGRAÇÃO
f4m_proxy.py Proxy reverso Flask na porta 8088. Rotas /hlsretry e /tsdownloader. Cache de chunks TS. Reconexão automática com User-Agent rotation. Flask · requests · urllib3 PROXY
dns.py Substitui socket.getaddrinfo() por resolver DNS customizado. Suporte a IPv4/IPv6. Cache em disco com TTL de 4h. socket · struct · UDP DNS REDE
doh.py DNS over HTTPS via Cloudflare (cloudflare-dns.com). Usado pelo F4M proxy para resolver hostnames de CDNs. HTTPS · Cloudflare DoH API REDE
proxy_http_scraper.py Scraper de lista pública de proxies HTTP com seleção aleatória e cache de 4h. Uso opcional. requests · JSON cache OPCIONAL

Fluxo de Reprodução — Canal Ao Vivo

Usuário seleciona canal
main.py mode='play' → play_item() detecta content_type='live'
Verifica F4MTESTER
is_f4m_installed() + check_f4m_server() → popup HLSRETRY ou TSDOWNLOADER
URL via F4M Proxy
http://127.0.0.1:8088/hlsretry?url=ENCODED_URL → Flask recebe a requisição
Proxy baixa e retransmite
DNSOverrideDoH() → requests.get() → rewrite_m3u8_urls() → stream_response() com cache de chunks
Kodi reproduz
ListItem com is_realtime_stream=true + buffer aumentado. Reconexão automática em falhas.
04 — Integrações

APIs e Integrações Externas

🎬

TMDB API v3

Busca automática de metadados para filmes, séries, temporadas e episódios. Também cruza programa atual do EPG para enriquecer canais ao vivo. Cache local de 7 dias.


api.themoviedb.org/3
📡

Xtream Codes API

API padrão de IPTV para autenticação, listagem de canais, VOD e séries. Suporta múltiplos servidores simultâneos com fallback automático.


player_api.php · xmltv.php
☁️

JSON Remoto de Servidores

Arquivo JSON hospedado em servidor próprio com lista de servidores, prioridades e flag debug_mode. Parser próprio sem json.loads (workaround bug Kodi).


portalbot.ebj.re/addon/server.json
🌐

Cloudflare DoH

DNS over HTTPS via cloudflare-dns.com para resolução segura de nomes no proxy F4M. Contorna bloqueios de DNS de ISPs e evita vazamento de consultas.


cloudflare-dns.com/dns-query

Formato do JSON Remoto de Servidores

// server.json — hospedado em servidor próprio { "debug_mode": false, // true = logs detalhados no Kodi "servers": [ { "id": 1, "name": "Servidor BR-1", "url": "http://servidor.exemplo.com:80", "username": "usuario", "password": "senha", "priority": 1, "enabled": true } ] }

Bug exclusivo do Kodi: O módulo json do Python nativo lança internamente um ValueError ao falhar o parse, mas no ambiente Kodi esse nome não está no escopo global, resultando em NameError: name 'ValueError' is not defined. A solução foi criar um parser JSON próprio usando apenas re e manipulação de string — sem nenhuma dependência do módulo json.

05 — Segurança

Modelo de Segurança

Proteção Implementação Onde
Controle de Logs Campo debug_mode no JSON remoto. Quando false, todos os logs LOGDEBUG são suprimidos — sem URLs, usuários, senhas ou IPs nos logs do Kodi. main.py · server_loader.py
Erros Genéricos Popups de erro ao usuário nunca expõem host, porta, usuário ou senha. Mensagens como "Servidor Offline" ou "Conectando no próximo..." main.py get_json()
Fallback Silencioso Troca de servidor automática sem revelar qual servidor estava em uso. Popup mostra apenas posição numérica (ex: 2/10). main.py try_next_server()
Silenciamento urllib3 Loggers do urllib3, requests e root definidos como CRITICAL, impedindo vazamento de URLs e IPs nas conexões HTTP. server_loader.py
DNS Seguro Uso de DoH (DNS over HTTPS) no proxy F4M para evitar vazamento de consultas DNS ao ISP. doh.py · f4m_proxy.py
Cache Criptografado Senhas dos servidores ficam em cache local apenas pelo TTL configurado. Não há transmissão de credenciais em texto além do protocolo Xtream padrão. server_loader.py
06 — Configuração

Guia de Configuração

⚙️

settings.xml

Configurações do add-on via interface Kodi: API key TMDB, idioma, cache, EPG on/off, proxy HTTP, F4MTESTER on/off, servidores manuais (até 5).

🌐

JSON Remoto

Hospede um arquivo server.json com a lista de servidores. Atualize quando quiser — os clientes baixam automaticamente ao expirar o cache (5 min padrão).

🔧

debug_mode

Mude "debug_mode": true no JSON para ver logs detalhados durante manutenção. Volte para false quando terminar — os clientes aplicam na próxima atualização do cache.

Configurações do settings.xml

Configuração Tipo Descrição Padrão
tmdb_enabled bool Ativa busca de metadados TMDB para filmes, séries e live false
tmdb_api_key string Chave da API TMDB (gratuita em themoviedb.org) —
tmdb_language select Idioma dos metadados retornados pelo TMDB pt-BR
enable_epg bool Ativa EPG/XMLTV nos canais ao vivo false
use_f4mtester bool Usa F4MTESTER proxy para canais ao vivo true
auto_fallback bool Fallback automático para próximo servidor em caso de falha true
proxy_http bool Usa proxy HTTP rotativo para requisições false
