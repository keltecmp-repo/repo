01 — Visão Geral

O que é o KelTec MP?

Add-on Kodi de última geração para consumo de conteúdo Xtream Codes. Inclui proxy de streaming nativo (KPN), autenticação VIP com JWT, metadados TMDB, EPG integrado, controle parental e gerenciamento multi-servidor totalmente remoto.

⚡

Proxy Nativo KPN — Sem F4M

Proxy de streaming totalmente interno, sem dependência de add-ons externos como F4M Tester ou KPN Tester. Três modos: HLS Bridge, HLS Retry e TS Downloader. Inicia automaticamente na porta 8088.

kpn.py — porta 8088
🔐

Autenticação VIP com JWT

Sistema completo de autenticação por token JWT. Login com usuário e senha, token salvo localmente com tolerância de fuso de 6h, validação online com cache de 120s. Painel web ADM com controle de dispositivos.

lib/vip_manager.py + painel PHP
📡

Multi-Servidor Remoto

18+ servidores gerenciados via JSON remoto, sem reinstalar o add-on. ServerLoader v5.0 com cache cifrado XOR+Base64, TTL configurável e fallback automático por prioridade.

ServerLoader v5.0
🎬

TMDB Nativo Completo

Dois módulos independentes: TMDB Helper enriquece metadados do servidor Xtream em background; TMDB Browser oferece catálogos completos com posters, sinopses, elenco e gêneros em pt-BR.

lib/tmdb_helper.py + lib/tmdb_browser.py
📺

EPG / XMLTV Automático

Guia eletrônico de programação com download automático, thread daemon de atualização a cada 5 minutos e índice de nomes com 444+ entradas para matching de canais. Não bloqueia a interface.

EPG_XML_PATH integrado
🔒

Controle Parental por PIN

Bloqueio de categorias adultas com PIN de 4+ dígitos. Detecção automática por palavra-chave, desbloqueio por sessão, proteção de configurações e interceptação em todos os pontos de acesso.

parental_menu + is_adult_category()
📊

Painel ADM Online

Painel web completo em PHP com visualização de dispositivos Online Agora em tempo real, limite de conexões simultâneas por usuário, expulsão de sessões, gestão de usuários VIP e logs de acesso.

keltec_panel/ — PHP + MySQL
🌐

DNS over HTTPS

Resolução DNS via Cloudflare DoH com cache de 4 horas. Contorna bloqueios de ISP, evita vazamento de consultas e garante resolução confiável em qualquer rede sem configuração adicional.

customdns — cloudflare-dns.com
📝

Histórico e Estatísticas

Registro automático de conteúdo assistido com histórico navegável, integração Trakt.tv para sincronização de watch history, ratings e watchlist, e painel de estatísticas pessoais.

lib/watch_history.py + lib/trakt_manager.py + lib/stats_manager.py

Principais Novidades — v2.1.8

REMOVIDO

F4M Tester — Não é mais necessário

O add-on externo F4M Tester foi completamente substituído pelo proxy nativo KPN integrado. Streams HLS, TS e M3U8 são processados internamente sem dependência de nenhum add-on externo. Quem tinha o F4M instalado pode removê-lo sem impacto.

NOVO

Proxy KPN Nativo — 3 Modos de Stream

HLS Bridge (stream contínuo sem rebobinamento), HLS Retry (por segmento clássico) e TS Downloader (streams TS puros). Seleção via popup ao reproduzir canal ao vivo. Correções de replay: heal reverso removido, ring buffer como fallback eliminado, served_urls com histórico permanente.

NOVO

Sistema VIP Completo com JWT e Painel ADM

Autenticação por usuário e senha com token JWT assinado. Painel PHP com gestão de usuários, planos, limite de dispositivos simultâneos, Online Agora em tempo real e expulsão remota de sessões. Logout automático limpa as 4 camadas de cache do Kodi.

NOVO

Limite de Dispositivos Simultâneos

Cada login VIP carrega max_devices do JWT. Ao tentar conectar além do limite, o Kodi exibe diálogo com lista de dispositivos ativos e bloqueia o login. O administrador pode expulsar sessões pelo painel.

NOVO

Online Agora — Painel em Tempo Real

Heartbeat a cada 60s reporta dispositivo, conteúdo assistido (TV ao Vivo, Filmes, Séries, Navegando), IP e horário de início. Contadores de dispositivos online, assistindo TV e VOD no topo do painel. Auto-refresh a cada 30s.

CORREÇÃO

Logout Limpa as 4 Camadas de Cache Automaticamente

Antes era necessário ir a outra ferramenta para limpar o cache do Kodi após logout. Agora o logout executa limpeza automática: cache cifrado do ServerLoader, arquivos .bin/.json do add-on, pasta temp do Kodi e propriedades de window em memória. Tudo com barra de progresso discreta.

CORREÇÃO

Device ID Permanente — Sem Duplicata no Painel

O arquivo keltec_device_id.bin agora está na lista de preservação do clear_cache, evitando que o mesmo dispositivo apareça duas vezes no Online Agora após a limpeza de cache.

CORREÇÃO

Cache TMDB com Escrita Atômica

O arquivo tmdb_cache.json agora usa escrita atômica via tempfile + os.replace() com fsync. Elimina corrupção do cache quando o Kodi é fechado no meio de uma escrita.

MELHORIA

Visualização Automática por Tipo de Conteúdo

Filmes, séries e temporadas abrem em modo Poster/Grade (500) automaticamente. Menus, categorias, TV ao vivo e episódios abrem em modo Lista (50). Sem necessidade de configuração manual.

SEGURANÇA

Token VIP com Tolerância de Fuso Unificada

A função jwt_verify() no painel e o vip_manager.py agora usam a mesma tolerância de 4-6h, evitando que diferenças de fuso horário causem logout inesperado.

02 — Proxy Nativo KPN

KPN — Proxy Interno de Streaming

O módulo KPN substitui completamente o F4M Tester e o KPN Tester externos. Roda na porta 8088 como thread daemon dentro do processo Kodi, sem nenhum add-on adicional necessário.

✅

F4M Tester e KPN Tester não são mais necessários

O KPN nativo do KelTec MP oferece os mesmos recursos (e mais) de forma integrada. Se você os tinha instalados, pode removê-los sem nenhum impacto. O KelTec MP detecta e usa seu próprio proxy automaticamente.

PROXY NATIVO KelTec Stream Proxy Porta 8088 · localhost

🌉 HLS Bridge

Baixa segmentos HLS sequencialmente e serve como stream TS contínuo numa única conexão TCP. Gerencia a playlist, prefetch de segmentos e poller de reconexão. Menor latência e maior estabilidade.

/hlsretry → hlsbridge interno

🔄 HLS Retry

Serve a playlist M3U8 reescrita para o Kodi gerenciar os requests de segmento. Compatível com todos os players. Retry automático por segmento com heal forward (+1) sem rebobinamento.

/hlsretry?url=

📦 TS Downloader

Para streams TS puros (não M3U8). Baixa e retransmite o fluxo TS diretamente. Se a URL for M3U8, redireciona automaticamente para o HLS Bridge. Ideal para servidores que entregam TS nativo.

/tsdownloader?url=
🛠️ Ao reproduzir um canal ao vivo, um popup permite escolher o modo. O KPN inicia automaticamente ao abrir o add-on — nenhuma configuração manual necessária. HLS Bridge recomendado para a maioria dos servidores.

Correções de Replay Aplicadas no KPN

🔴

Heal Reverso Removido

O heal de segmento só avança (+1). A direção -2 (segmento anterior) foi removida — servir dados do passado ao Kodi causa rebobinamento automático.

🔴

Ring Buffer Fallback Removido

Chunks antigos do ring buffer não são mais servidos como fallback final. Na falha total, retorna 502 e o Kodi reconecta do início — sem replay.

🔴

served_urls Permanente

O histórico de URLs já servidas no HLS Bridge nunca é limpo por intersection_update. Segmentos que saem e voltam na playlist não são re-servidos.

// Lógica de seleção ao reproduzir canal ao vivo if use_KPN and KPN_running(): popup → [HLS Bridge | TS Downloader | HLS Retry] choice == 0 (HLS Bridge): url = KPN_HLSRETRY + quote(convert_to_m3u8(url)) → proxy roteia internamente para hlsbridge choice == 1 (TS Downloader): url = KPN_TSDOWNLOADER + quote(url_sem_m3u8) choice == 2 (HLS Retry): url = KPN_HLSRETRY + quote(url_m3u8) else: reprodução direta sem proxy
03 — Sistema VIP

Autenticação VIP e Painel ADM

Sistema completo de autenticação com JWT, controle de dispositivos simultâneos, painel web de administração em tempo real e integração com o proxy de sessão.

Fluxo de Autenticação

01

02

03

Token Salvo

04

servers.php

Token enviado para API remota que retorna lista de servidores desbloqueados

05

Sessão Ativa

Heartbeat a cada 60s mantém presença no Online Agora do painel

Controle de Dispositivos Simultâneos

🔢

Limite por Usuário

Cada conta VIP tem limite configurável de dispositivos simultâneos. O campo max_devices vem embutido no JWT.

🔍

Verificação no Login

Antes de iniciar sessão, session.php conta dispositivos ativos. Se atingido o limite, exibe os ativos e bloqueia.

💓

Heartbeat 60s

Cada dispositivo envia heartbeat a cada 60s com device_id único persistente. TTL de 240s no servidor.

⛔

Expulsão Remota

Admin expulsa pelo painel → kicked_sessions → próximo heartbeat do Kodi chama PlayerControl(Stop) e faz logout.

📱

Device ID Permanente

keltec_device_id.bin nunca é apagado na limpeza de cache. Garante que o mesmo dispositivo sempre aparece como uma linha no painel.

🖥️

Detecção de Plataforma

Android TV Box identificado via System.OSVersionInfo em vez de platform.system() que retornava "Linux" incorretamente.

Painel ADM — Funcionalidades

🟢

Online Agora

Visualização em tempo real de todos os dispositivos conectados. Mostra usuário, plano, device, IP, conteúdo assistido (TV/VOD/Navegando), último heartbeat e limite atingido. Auto-refresh 30s.

👥

Gestão de Usuários

Criar, editar e excluir usuários VIP. Campos: usuário, senha, nome, WhatsApp/telefone (link direto para WhatsApp), e-mail, plano, expiração, limite de dispositivos e observações.

🔍

Busca e Filtros

Busca por usuário, nome completo, telefone ou e-mail em tempo real. Filtros por plano: Teste, Mensal, Trimestral, Anual e Vitalício. Contador de resultados.

☑️

Ações em Lote

Seleção múltipla de usuários com "selecionar todos". Ações: ativar, desativar ou excluir em lote. Confirmação antes de excluir.

📋

Logs de Acesso

Histórico completo de autenticações com usuário, resultado (sucesso/falha), IP e horário. Exibição direta sem conversão de fuso horário — MySQL configurado em America/Sao_Paulo.

⏱️

Extender Acesso

Botão "+Dias" por usuário abre modal para adicionar dias ao acesso atual. Atualiza automaticamente a data de expiração sem editar o usuário completo.

Logout Automático com 4 Camadas de Cache

Novo em v2.1.8: ao fazer logout no add-on, as 4 camadas de cache do Kodi são limpas automaticamente com barra de progresso discreta — sem necessidade de ferramenta externa.

Camada 1: Cache cifrado do ServerLoader (keltec_srv_data.bin)
Camada 2: Arquivos .bin e _cache.json da pasta do add-on (preserva device_id, token VIP e favoritos)
Camada 3: Pasta special://temp/plugin.video.keltecmp-xtream/
Camada 4: Window properties em memória do Kodi (keltec_vip_status, keltec_server_name, etc.)
04 — TMDB · Metadados

Integração TMDB Profunda

Dois módulos independentes: TMDB Helper enriquece metadados do servidor Xtream em segundo plano; TMDB Browser oferece catálogos completos resolvidos no servidor.

🔍

TMDB Helper — Enriquecimento

Busca automática de metadados para itens do servidor Xtream: poster, fanart, sinopse, nota, elenco, diretor e gênero. Thread de background, sem travar a interface.

lib/tmdb_helper.py
🗂️

TMDB Browser — Catálogo

Navegação independente do servidor: listas populares, avaliação, gêneros e busca. Resolve automaticamente qual stream do servidor corresponde a cada título TMDB.

lib/tmdb_browser.py
⚡

Cache Atômico Dois Níveis

Dicionário em memória para a sessão atual + JSON em disco com TTL de 7 dias. Escrita atômica via tempfile+fsync+os.replace() — sem corrupção se o Kodi fechar no meio da escrita.

🎯

Matching Jaccard

Normaliza e compara títulos do servidor (com sufixos [Dual], [4K], (2026)) com títulos TMDB. Agrupa variantes do mesmo título evitando chamadas duplicadas à API.

🌍

pt-BR Nativo

Todos os metadados — sinopse, título, gêneros, elenco — retornados em pt-BR. Configurável para qualquer locale IANA nas opções do add-on.

▶️

Página de Detalhes

Trailer YouTube, sinopse completa, ficha técnica, elenco com personagens, filmes semelhantes e coleções. Trailer executa via PlayMedia() sem passar pelo proxy.

Requisito: Configure sua API Key TMDB gratuita em themoviedb.org/settings/api e insira nas configurações do add-on. Sem a chave, TMDB Browser e Helper ficam desativados automaticamente.

06 — Controle Parental

Sistema de Controle Parental

Proteção completa de conteúdo adulto com PIN de 4+ dígitos, detecção automática por palavras-chave e desbloqueio por sessão sem redigitar PIN a cada categoria.

🔑

PIN por Sessão

Uma vez digitado o PIN correto, o acesso adulto fica liberado para toda a sessão Kodi atual. Arquivo temporário de sessão em /tmp — reiniciar o Kodi exige nova autenticação.

🤖

Detecção Automática

is_adult_category() analisa o nome da categoria por palavras-chave (adulto, 18+, xxx, etc.) e intercepta automaticamente — sem configuração manual por categoria.

🛡️

Proteção Total

Interceptação em todos os pontos: entrada em categorias live/vod/séries adultas, reprodução direta (is_adult=true na URL) e acesso ao próprio menu de configurações parentais.

🔐

PIN Seguro

Para desativar o controle parental ou alterar o PIN, o PIN atual é sempre exigido — evita burla por crianças. PIN armazenado em settings.xml do Kodi.

// Opções disponíveis em mode=parental_menu [1] Ativar / Desativar → toggle — desativar exige PIN atual [2] Alterar PIN → exige PIN atual + confirmação do novo [3] Desbloquear Sessão → exige PIN → libera até reiniciar Kodi [4] Bloquear Sessão → força PIN na próxima categoria adulta

08 — Segurança

Modelo de Segurança Multicamada

01

🔐 JWT Assinado com HMAC-SHA256

Token JWT gerado com chave secreta API_SECRET. Tolerância de fuso de 4h no servidor (functions.php) e 6h no cliente (vip_manager.py). Consistência entre todas as rotas de verificação.

02

🔒 Cache de Servidores Cifrado XOR+Base64

Cache de servidores em keltec_srv_data.bin com chave de 16 bytes. Descriptografado apenas em memória. Remove automaticamente versões JSON em texto puro se encontradas.

03

🛡️ URLs de Stream Nunca Aparecem nos Logs

Filtro _KeltecURLFilter em todos os loggers Python. Intercepta http://, https://, username=, .m3u8, IPs e padrões de stream antes de chegar ao kodi.log.

04

🔑 Proxy Token — Streams Ocultos

Cada URL de stream é substituída por token XOR+Base64 antes de passar ao Kodi. O proxy interno decodifica e retransmite. O kodi.log nunca registra URLs reais de servidor.

05

📱 Controle de Dispositivos por JWT

max_devices embutido no JWT — não pode ser alterado pelo cliente. Verificação server-side a cada login. Expulsão remota via painel com notificação no dispositivo afetado.

06

🦠 Detecção de Add-ons Parasitas

Na inicialização, verifica se add-ons listados em _PARASITAS_IDS usam as mesmas credenciais. Se detectado, alerta o usuário com recomendações de ação.

07

🌐 DNS over HTTPS Cloudflare

Resoluções DNS via cloudflare-dns.com/dns-query com cache de 4 horas. Contorna bloqueios de ISP e evita vazamento de consultas DNS em qualquer rede.

08

🔕 Loggers HTTP Silenciados

urllib3, requests, werkzeug e root configurados como CRITICAL e desabilitados. Nenhuma URL de conexão HTTP vaza para logs do sistema.

10 — Configuração

Guia de Configuração

Não é necessário configurar proxy externo. O add-on inclui o proxy KPN nativo integrado. Basta instalar o add-on e autenticar com suas credenciais VIP.

Configurações Principais (settings.xml)

Formato do JSON Remoto de Servidores

// Hospedado remotamente — atualizações sem reinstalar o add-on { "debug_mode": false, "version": "2.1.8", "servers": [ { "id": 1, "name": "Servidor BR-1", "url": "http://servidor.exemplo.com:80", "username": "usuario_vip", // gerenciado pelo painel "password": "senha_vip", "priority": 1, // menor = maior prioridade "enabled": true } // ... até 18 servidores simultâneos ] }
Cache de servidores: o ServerLoader mantém cache cifrado local com TTL de 1h. Após trocar de servidor pelo seletor, o cache é limpo e os servidores são rebuscados da API remota imediatamente — sem necessidade de reiniciar o add-on.
