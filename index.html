01 — Visão Geral

O que é o KelTec MP?

📡

Multi-Servidor Xtream

Suporte a 13+ servidores simultâneos com fallback automático. Troca transparente de servidor quando um falha — sem interrupção para o usuário. Prioridade configurável por servidor.

🎬

Catálogo TMDB Nativo

Browser próprio integrado ao TMDB. Navegue por filmes e séries com metadados completos, filtros por gênero, busca por título e resolução automática no servidor Xtream.

🔒

Controle Parental PIN

Sistema completo de controle parental com PIN de 4+ dígitos. Proteção por sessão — desbloqueio único por sessão Kodi. Detecção automática de categorias adultas por palavra-chave.

🛡️

Proxy Local Seguro

Servidor HTTP interno na porta 8089. Retransmite streams com User-Agent correto sem expor URLs originais no kodi.log. Tokenização XOR+Base64 para cada stream.

📺

EPG / XMLTV

Guia eletrônico de programação com atualização automática em background a cada 5 minutos. Thread daemon independente que não bloqueia a interface do usuário.

🎭

Trailers YouTube

Integração com plugin.video.youtube para reprodução de trailers oficiais direto da tela de detalhes TMDB. Execução correta via PlayMedia — sem passar pelo proxy local.

📄

Paginação Inteligente

Catálogos com paginação automática. Botões "Página Anterior" e "Próxima Página" com indicação de progresso (ex: 3/47). Limite de itens por página configurável.

🌐

DNS Customizado

DNS over HTTPS via Cloudflare com cache de 4 horas. Contorna bloqueios de ISP, evita vazamento de consultas DNS e garante resolução confiável de nomes em qualquer rede.

☁️

Servidores Remotos

JSON de servidores hospedado remotamente com cache local protegido. Atualizações sem reinstalação do add-on. Parser JSON próprio sem dependências do módulo padrão do Kodi.

02 — TMDB · Metadados

Integração TMDB Profunda

O add-on integra dois módulos TMDB independentes: o TMDB Helper enriquece metadados de itens do servidor Xtream em tempo real; o TMDB Browser oferece catálogos completos da base TMDB resolúveis no servidor.

🔍

TMDB Helper — Enriquecimento

Busca automática de metadados para cada item do servidor Xtream. Poster, fanart, sinopse, classificação, elenco, diretor, duração e gênero carregados em thread de segundo plano sem travar a interface.


lib/tmdb_helper.py
🗂️

TMDB Browser — Catálogo

Módulo de navegação TMDB independente do servidor. Listas populares, avaliação, gêneros, busca por texto. Resolve automaticamente qual stream do servidor corresponde a cada título TMDB.


lib/tmdb_browser.py
⚡

Cache em Memória + Disco

Cache TMDB em dois níveis: dicionário em memória para a sessão atual (acesso imediato) e arquivos JSON em disco com TTL configurável (padrão 7 dias). Thread-safe com RLock.

🌍

Idioma Configurável

Todos os metadados — sinopse, título, gêneros, elenco — retornados no idioma configurado nas preferências. Padrão pt-BR com suporte a qualquer locale IANA.

🎯

Similaridade Jaccard

Algoritmo de normalização e similaridade para casar títulos do servidor (com sufixos como [Dual], [4K], (2026)) com os títulos exatos do TMDB. Limiar configurável, sim=1.00 indica match perfeito.

🔄

Reutilização de Detalhes

Variantes do mesmo título (ex: "Filme (2026)" e "Filme (2026) [Dual]") compartilham o mesmo objeto de detalhes TMDB. Evita chamadas duplicadas à API e reduz uso de cota.

Requisito: Configure sua API Key TMDB gratuita em themoviedb.org/settings/api e insira nas configurações do add-on. Sem a chave, o TMDB Browser e o TMDB Helper ficam desativados automaticamente.

06 — Controle Parental

Sistema de Controle Parental

O controle parental é um sistema completo de bloqueio por PIN integrado diretamente ao fluxo de navegação. Detecta categorias adultas automaticamente e protege conteúdo 18+ sem configuração manual por categoria.

Proteção por PIN

PIN de 4+ dígitos configurado pelo usuário. Armazenado em settings.xml do Kodi. Para desativar o controle parental, o PIN atual é exigido — protege contra burla por crianças.

Desbloqueio por Sessão

Uma vez digitado o PIN correto, o acesso adulto fica liberado para toda a sessão Kodi atual. Sem necessidade de redigitar a cada categoria. Reiniciar o Kodi exige nova autenticação.

Detecção Automática

Função is_adult_category() analisa o nome da categoria por palavras-chave (ex: "adulto", "18+", "xxx"). Intercepta automaticamente ao entrar na categoria.

Bloqueio Granular

Proteção em todos os pontos: entrada em categorias (live, vod, séries), reprodução direta de conteúdo marcado como adulto, e acesso ao menu de configurações parentais.

Menu de Controle Parental

// Opções disponíveis no menu parental_menu [1] Ativar / Desativar → toggle com exigência de PIN para desativar [2] Alterar PIN → exige PIN atual + confirmação do novo [3] Desbloquear Sessão → exige PIN → libera acesso até reiniciar Kodi [4] Bloquear Sessão → força novo PIN na próxima categoria adulta [5] Voltar → fecha menu sem alterações

Proxy KelTec Interno — Porta 8089

🔁

Thread Daemon

Servidor HTTP iniciado em thread daemon no boot do add-on. Roda dentro do processo Kodi — sem processos externos. Nome da thread: KelTecProxy.

🔑

Tokenização XOR

Cada URL de stream é substituída por um token gerado via XOR+Base64. O proxy mapeia token → URL original internamente. URL real nunca aparece em logs do Kodi.

🎬

Filmes e Séries

Filmes e séries usam o proxy KelTec por padrão (use_keltec_proxy=true). Canais live usam F4M (porta 8088). Pode ser desabilitado nas configurações.

🚫

Bypass plugin://

URLs plugin:// (como trailers YouTube) são detectadas antes do proxy e executadas via PlayMedia() diretamente.

// Lógica de seleção de proxy em play_item() if url_clean.startswith('plugin://'): → URL de plugin externo (YouTube, etc.) xbmcplugin.setResolvedUrl(handle, False, ListItem()) xbmc.executebuiltin(f'PlayMedia({url_clean})') return elif content_type in ['movie', 'series']: if use_proxy == 'true' and _keltec_proxy_running(): url_final = _keltec_proxy_url(url_clean) // token XOR else: li.setProperty('customHeaders', f'User-Agent={UA}') elif content_type == 'live': → Usa F4MTESTER (porta 8088) ou player direto

ServerLoader v5.0

☁️

JSON Remoto

Arquivo JSON hospedado remotamente com lista de servidores. Baixado e cacheado localmente. Parser próprio sem dependência do módulo json (workaround bug Kodi).

⚡

Cache Criptografado

Cache salvo em keltec_srv_data.bin com cifra XOR+Base64. TTL configurável. Remove automaticamente versões legíveis antigas.

08 — Segurança

Modelo de Segurança Multicamada

01

🔐 Cache Criptografado XOR+Base64

Cache de servidores salvo em formato binário cifrado. Chave de 16 bytes _CACHE_KEY configurável. Descriptografado apenas em memória durante a sessão. Remove versão JSON em texto puro se encontrada.

02

🚫 Bloqueio Total de URLs em Logs

Filtro _KeltecURLFilter instalado em todos os loggers Python (root, werkzeug, urllib3, requests). Intercepta qualquer mensagem contendo http://, https://, username=, .m3u8, IPs e padrões de stream antes de chegar ao kodi.log.

03

🛡️ Proxy Token — Oculta URLs de Stream

Cada URL de stream real é substituída por um token XOR+Base64 antes de ser passada ao Kodi. O kodi.log registra apenas o token — nunca a URL do servidor. O proxy interno decodifica e retransmite.

04

🦠 Detecção de Add-ons Parasitas

Na inicialização, verifica se add-ons listados em _PARASITAS_IDS estão usando as mesmas credenciais (URL + usuário). Se detectado, alerta o usuário com recomendações. Não modifica dados automaticamente.

05

🌐 DNS over HTTPS

Módulo customdns redireciona resoluções DNS via Cloudflare DoH (cloudflare-dns.com/dns-query) com cache de 4 horas. Contorna bloqueios de ISP e evita vazamento de consultas DNS.

06

🔕 Silenciamento de Loggers HTTP

Loggers urllib3, requests, werkzeug e root configurados como CRITICAL e desabilitados. Nenhuma URL de conexão HTTP vaza para os logs do sistema.

07

🔒 Controle Parental por PIN

PIN armazenado em settings.xml (settings do Kodi). Verificação obrigatória para desativar proteção, alterar PIN ou entrar em categorias adultas. Sessão de acesso adulto não persiste entre reinicializações.

08

📋 Erros Genéricos ao Usuário

Mensagens de erro nunca expõem host, porta, usuário ou senha. Popups mostram apenas "Servidor Offline" ou "Conectando no próximo servidor (2/13)" — sem informação técnica sensível.

Aviso de Segurança: O sistema de detecção de parasitas verifica apenas add-ons da lista _PARASITAS_IDS. Se suspeitar de uso não autorizado das suas credenciais por outros add-ons, altere sua senha no painel do servidor Xtream imediatamente.
09 — Configuração

Guia de Configuração

⚙️

settings.xml

Configurações do add-on via interface nativa do Kodi. Inclui credenciais TMDB, opções de proxy, controle parental, EPG, F4MTESTER e servidores manuais (até 5 entradas).

🌐

JSON Remoto de Servidores

Hospede um arquivo server.json com a lista de servidores. Atualizações aplicadas automaticamente ao expirar o cache sem reinstalar o add-on.

🔧

debug_mode

Ative "debug_mode": true no JSON remoto para logs detalhados durante manutenção. Clientes aplicam na próxima atualização do cache.

Formato do JSON Remoto de Servidores

// Hospedado em servidor próprio — atualize quando quiser { "debug_mode": false, // true = logs detalhados no Kodi "servers": [ { "id": 1, "name": "Servidor BR-1", "url": "http://servidor.exemplo.com:80", "username": "usuario", "password": "senha", "priority": 1, // menor = maior prioridade "enabled": true } // ... até 13 servidores ] }

Bug exclusivo do Kodi: O módulo json do Python lança ValueError ao falhar o parse, mas no ambiente Kodi esse nome pode não estar no escopo global, resultando em NameError: name 'ValueError' is not defined. A solução é um parser JSON próprio usando apenas re e manipulação de string — sem dependências externas.
